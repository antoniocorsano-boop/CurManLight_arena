# CML-607 — Runtime View Rendering Recovery

## Causa

`src/main.tsx` monta `App` dentro `BrowserRouter`, ma non usa il router definito in `src/routes/index.tsx`. `App.tsx` prepara `AppViewsLayerProps`, ma nell'area principale rende solo `<Outlet />` (che restituisce `null` senza route attive). Le viste reali non sono mai raggiungibili nella build.

## Baseline

- Branch: `main` = `origin/main` = `be3ddb9`
- Commit HEAD iniziale: `be3ddb9`
- Branch: `fix/cml-607-runtime-view-rendering`

## Impatto

- L'area contenuto principale (`#main-content`) resta vuota
- Nessuna vista (Curriculum, Documenti, Progetta, Classroom, etc.) è visualizzabile
- Il blocco è preesistente rispetto a CML-606
- CML-606 è sospeso finché questo non viene risolto

## Strategia scelta

Sostituzione minima: `<Outlet />` → `<AppViewsLayer {...appViewsLayerProps} />`

- Mantenuto `BrowserRouter` in `src/main.tsx`
- Mantenuta shell, header, sidebar e navigazione
- `AppViewsLayer` già esistente e già riceve tutte le props necessarie
- Nessuna nuova route, nessuna nuova dipendenza

## Alternativa non adottata

`RouterProvider` con `createBrowserRouter` (in `src/routes/index.tsx`). Sarebbe richiesta una migrazione completa del routing, fuori perimetro per CML-607.
