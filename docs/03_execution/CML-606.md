# CML-606 — UI System Foundation and Documents Pilot

## Obiettivo di prodotto
"Il docente può usare CurManLight attraverso un'interfaccia coerente, leggibile e prevedibile, nella quale azioni, contenuti, stati e fonti hanno una gerarchia visiva stabile."

## Baseline
- main = origin/main = be3ddb9
- test: 222/222 PASS (pre-CML-606)
- build: 788 KB, verde
- TypeScript: 143 errori pre-esistenti

## Audit UI
- 757 usi di font-size arbitrario (molti sotto 10px) — problema critico accessibilità
- 464 usi di `font-black` — interfaccia aggressiva
- Triplice palette cromatica (indigo/purple/violet) non intenzionale
- Border-radius incoerenti (rounded-xl, 2xl, 3xl interchangeabili)
- Componenti Card/Badge esistenti ma ignorati
- Ombre colorate non standard
- 263 pattern card inline
- 525 usi di `uppercase`

## Soluzione
Introduzione di un sistema UI semantico con token CSS, componenti fondamentali e migrazione della vista Documenti/Esportazioni come pilota.
